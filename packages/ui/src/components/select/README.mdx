# Select Component

## Overview

The Select component is a fully-featured, accessible dropdown select implementation for React applications. It provides a customizable and flexible way to create dropdown menus with various styling options and behaviors.

## Features

- Fully accessible, following WAI-ARIA guidelines
- Keyboard navigation support
- Customizable styling with Tailwind CSS
- Support for grouped options
- Typeahead search functionality
- Scrollable dropdown with custom scroll buttons
- Support for disabled options
- Customizable positioning (popper or item-aligned)
- Form integration with native select fallback

## Installation

To use the Select component, ensure you have the necessary dependencies installed in your project:

```bash
npm install @jsandy/ui
```

## Usage

Here's a basic example of how to use the Select component:

```jsx
import {
  Select,
  SelectTrigger,
  SelectValue,
  SelectContent,
  SelectItem,
} from '@jsandy/ui/select';

function MySelect() {
  return (
    <Select>
      <SelectTrigger>
        <SelectValue placeholder="Select an option" />
      </SelectTrigger>
      <SelectContent>
        <SelectItem value="option1">Option 1</SelectItem>
        <SelectItem value="option2">Option 2</SelectItem>
        <SelectItem value="option3">Option 3</SelectItem>
      </SelectContent>
    </Select>
  );
}
```

## Components

### Select

The root component that manages the overall state of the select.

#### Props

- `value`: string - The controlled value of the select.
- `defaultValue`: string - The default value for uncontrolled usage.
- `onValueChange`: (value: string) => void - Callback when the value changes.
- `open`: boolean - Controls the open state of the select.
- `defaultOpen`: boolean - The default open state for uncontrolled usage.
- `onOpenChange`: (open: boolean) => void - Callback when the open state changes.
- `dir`: 'ltr' | 'rtl' - The text direction.
- `name`: string - The name of the select for form submission.
- `autoComplete`: string - The autocomplete attribute.
- `disabled`: boolean - Whether the select is disabled.
- `required`: boolean - Whether the select is required.

### SelectTrigger

The button that toggles the select dropdown.

### SelectValue

Displays the selected value or placeholder.

### SelectContent

The dropdown menu containing the options.

### SelectItem

An individual selectable option.

### SelectGroup

A group of related options.

### SelectLabel

A label for a group of options.

### SelectSeparator

A visual separator between items or groups.

## Accessibility

The Select component follows WAI-ARIA guidelines for accessibility:

- Proper ARIA roles and attributes are automatically applied.
- Keyboard navigation is fully supported (arrow keys, Home, End, typeahead).
- Focus management ensures a good experience for keyboard and screen reader users.

## Customization

The Select component and its subcomponents accept className props for custom styling. They use Tailwind CSS classes by default, which can be overridden or extended.

## Best Practices

1. Always provide a clear label for the select, either using a visible label or aria-label.
2. Group related options using SelectGroup and SelectLabel for better organization.
3. Use disabled options sparingly and provide clear visual indication.
4. Consider the number of options and use scrollable content when necessary.
5. Provide a meaningful placeholder to guide users in their selection.
