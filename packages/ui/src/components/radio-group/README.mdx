# Radio and RadioGroup Components

## Overview

The Radio and RadioGroup components are part of the @jsandy/ui package, providing accessible and customizable radio button functionality for React applications. These components offer a robust solution for creating single-selection inputs, with support for form integration, keyboard navigation, and custom styling.

## Features

- Accessible radio button and radio group implementations
- Support for controlled and uncontrolled usage
- Keyboard navigation within radio groups
- Form integration with native radio input behavior
- Customizable styling and indicators
- TypeScript support

## Installation

To use the Radio and RadioGroup components, ensure you have the @jsandy/ui package installed in your project:

```bash
npm install @jsandy/ui
```

## Usage

Here's a basic example of how to use the RadioGroup and RadioGroupItem components:

```jsx
import { RadioGroup, RadioGroupItem } from '@jsandy/ui/radio-group';

function MyForm() {
  return (
    <RadioGroup defaultValue="option1">
      <RadioGroupItem value="option1" id="option1">
        Option 1
      </RadioGroupItem>
      <RadioGroupItem value="option2" id="option2">
        Option 2
      </RadioGroupItem>
      <RadioGroupItem value="option3" id="option3">
        Option 3
      </RadioGroupItem>
    </RadioGroup>
  );
}
```

## API

### Radio

The base Radio component for creating individual radio buttons.

#### Props

- `checked`: boolean - Indicates whether the radio button is checked.
- `required`: boolean - Specifies if the radio button is required in a form.
- `disabled`: boolean - Disables the radio button when true.
- `value`: string - The value associated with the radio button.
- `onCheck`: () => void - Callback function triggered when the radio button is checked.
- `name`: string - The name of the radio button for form submission.

### RadioGroup

A component for grouping related radio buttons.

#### Props

- `name`: string - The name of the radio group for form submission.
- `required`: boolean - Specifies if selecting a radio button in the group is required.
- `disabled`: boolean - Disables the entire radio group when true.
- `dir`: 'ltr' | 'rtl' - Specifies the text direction.
- `orientation`: 'horizontal' | 'vertical' - Determines the layout orientation of the radio buttons.
- `loop`: boolean - Enables keyboard navigation to loop from last to first item and vice versa.
- `defaultValue`: string - The default selected value for uncontrolled usage.
- `value`: string - The currently selected value for controlled usage.
- `onValueChange`: (value: string) => void - Callback function triggered when the selected value changes.

### RadioGroupItem

An individual radio button to be used within a RadioGroup.

#### Props

- `value`: string - The value associated with the radio button item.
- `disabled`: boolean - Disables this specific radio button item when true.

## Accessibility

The Radio and RadioGroup components are built with accessibility in mind:

- Proper ARIA attributes are automatically applied.
- Keyboard navigation is supported within radio groups.
- Focus management is handled for ease of use with keyboard and screen readers.

## Customization

Both Radio and RadioGroup components accept className props for custom styling. The RadioGroupIndicator component can be used to create custom visual indicators for the radio buttons.

## Best Practices

1. Always provide a label or aria-label for each radio button for accessibility.
2. Use RadioGroup when you have multiple related options where only one can be selected.
3. Consider using a default selected option when appropriate.
4. Ensure sufficient spacing between radio buttons for easy interaction.
5. Use clear and concise labels for each option.

## Examples

### Controlled RadioGroup

```jsx
function ControlledRadioGroup() {
  const [value, setValue] = React.useState('option1');

  return (
    <RadioGroup value={value} onValueChange={setValue}>
      <RadioGroupItem value="option1" id="controlled1">
        Option 1
      </RadioGroupItem>
      <RadioGroupItem value="option2" id="controlled2">
        Option 2
      </RadioGroupItem>
    </RadioGroup>
  );
}
```

### RadioGroup with Custom Indicator

```jsx
function CustomRadioGroup() {
  return (
    <RadioGroup>
      <RadioGroupItem value="custom1" id="custom1">
        <RadioGroupIndicator />
        Custom Option 1
      </RadioGroupItem>
      <RadioGroupItem value="custom2" id="custom2">
        <RadioGroupIndicator />
        Custom Option 2
      </RadioGroupItem>
    </RadioGroup>
  );
}
```
