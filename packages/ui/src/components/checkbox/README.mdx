# Checkbox Component

The Checkbox component is a customizable and accessible way to allow users to make binary choices or select multiple options from a list. It combines a hidden native checkbox input with a styled visual representation, ensuring both aesthetics and functionality.

## Features

- Customizable appearance through CSS classes
- Accessible, using a native checkbox input
- Visual indicator for checked state
- Support for indeterminate state
- Integrates with form controls
- Supports standard input attributes like disabled, required, etc.

## Installation

```bash
npm install @jsandy/ui
```

## Usage

```tsx
import { Checkbox } from '@jsandy/ui/checkbox';

function MyComponent() {
  return (
    <div>
      <Checkbox id="terms" />
      <label htmlFor="terms">I agree to the terms and conditions</label>
    </div>
  );
}
```

## Props

| Prop            | Type     | Default         | Description                                             |
| --------------- | -------- | --------------- | ------------------------------------------------------- | ------------------------------------------ |
| checked         | boolean  | 'indeterminate' | undefined                                               | Controls the checked state of the checkbox |
| defaultChecked  | boolean  | 'indeterminate' | undefined                                               | Sets the initial checked state             |
| required        | boolean  | false           | Specifies if the checkbox is required in a form         |
| disabled        | boolean  | false           | Disables the checkbox                                   |
| value           | string   | 'on'            | The value associated with the checkbox                  |
| onCheckedChange | function | undefined       | Callback function called when the checked state changes |
| name            | string   | undefined       | The name of the checkbox when used in a form            |

All other props are passed directly to the underlying button element.

## Customization

You can customize the appearance of the Checkbox component by passing additional classes through the `className` prop:

```tsx
<Checkbox className="h-6 w-6 rounded-full" />
```

## Accessibility

The Checkbox component is designed with accessibility in mind:

- It uses a native checkbox input for proper form integration and screen reader support
- The visual checkbox is implemented using `role="checkbox"` for proper semantics
- It supports keyboard navigation and activation
- The component handles aria-checked states, including the indeterminate state

## Best Practices

- Always provide a label for the checkbox, either using the `htmlFor` attribute on a `<label>` element or by wrapping the Checkbox in a `<label>`
- Group related checkboxes together, often within a `fieldset` with a `legend`
- Use clear and concise labels to describe the option
- Consider using the indeterminate state for parent checkboxes in hierarchical selections

## Examples

### Basic Checkbox

```tsx
<Checkbox id="newsletter" />
<label htmlFor="newsletter">Subscribe to newsletter</label>
```

### Disabled Checkbox

```tsx
<Checkbox id="disabled-option" disabled />
<label htmlFor="disabled-option">Unavailable option</label>
```

### Checkbox with Indeterminate State

```tsx
const [checked, setChecked] = React.useState('indeterminate');

<Checkbox
  id="parent"
  checked={checked}
  onCheckedChange={setChecked}
/>
<label htmlFor="parent">Parent option</label>
```

### Checkbox Group

```tsx
<fieldset>
  <legend>Choose your toppings:</legend>
  <div>
    <Checkbox id="cheese" name="toppings" value="cheese" />
    <label htmlFor="cheese">Extra cheese</label>
  </div>
  <div>
    <Checkbox id="pepperoni" name="toppings" value="pepperoni" />
    <label htmlFor="pepperoni">Pepperoni</label>
  </div>
  <div>
    <Checkbox id="mushrooms" name="toppings" value="mushrooms" />
    <label htmlFor="mushrooms">Mushrooms</label>
  </div>
</fieldset>
```

By following these guidelines and examples, you can effectively use and customize the Checkbox component in your application to create intuitive and accessible user interfaces.
