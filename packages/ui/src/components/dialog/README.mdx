# Dialog Component

The Dialog component is a flexible and accessible modal dialog that interrupts the user with important content and expects a response. It's designed to focus the user's attention on critical information or actions without losing context of the underlying page.

## Features

- Controlled and uncontrolled usage
- Customizable trigger element
- Accessible keyboard navigation and focus management
- Overlay background for visual separation
- Smooth enter/exit animations
- Nested dialog support
- Customizable content, title, and description components

## Installation

```bash
npm install @jsandy/ui
```

## Usage

```tsx
import {
  Dialog,
  DialogTrigger,
  DialogContent,
  DialogHeader,
  DialogFooter,
  DialogTitle,
  DialogDescription,
} from '@jsandy/ui/dialog';
import { Button } from '@jsandy/ui/button';

function MyDialog() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="outline">Open Dialog</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Are you sure?</DialogTitle>
          <DialogDescription>
            This action cannot be undone. This will permanently delete your account and remove your
            data from our servers.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button type="submit">Confirm</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Components

### Dialog

The main wrapper component that provides context for its children.

### DialogTrigger

The button that opens the dialog.

### DialogContent

The component that contains the content of the dialog.

### DialogHeader

A container for grouping the dialog title and description.

### DialogFooter

A container for grouping the dialog footer content, typically action buttons.

### DialogTitle

The title of the dialog.

### DialogDescription

The description of the dialog, providing more context about its purpose or content.

### DialogClose

A button that closes the dialog. It's automatically rendered within DialogContent, but can be used for additional close triggers if needed.

## Customization

### Custom Trigger

You can customize the trigger by using the `asChild` prop:

```tsx
<DialogTrigger asChild>
  <Button variant="outline">Custom Trigger</Button>
</DialogTrigger>
```

### Styling

You can apply custom styles to each component:

```tsx
<DialogContent className="bg-primary text-primary-foreground">
  <DialogHeader>
    <DialogTitle className="text-2xl font-bold">Custom Styled Dialog</DialogTitle>
    <DialogDescription className="text-primary-foreground/70">
      This dialog has custom styling applied.
    </DialogDescription>
  </DialogHeader>
  {/* Dialog content */}
</DialogContent>
```

## Accessibility

The Dialog component follows accessibility best practices:

- Uses `role="dialog"` and `aria-modal="true"`
- Moves focus to the dialog when opened and restores focus when closed
- Traps focus within the dialog when open
- Closes the dialog when the escape key is pressed
- Provides `aria-labelledby` and `aria-describedby` for screen readers

## Examples

### Basic Dialog

```tsx
<Dialog>
  <DialogTrigger asChild>
    <Button variant="outline">Open Dialog</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Are you absolutely sure?</DialogTitle>
      <DialogDescription>
        This action cannot be undone. This will permanently delete your account and remove your data
        from our servers.
      </DialogDescription>
    </DialogHeader>
    <DialogFooter>
      <Button type="submit">Confirm</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

### Dialog with Form

```tsx
<Dialog>
  <DialogTrigger asChild>
    <Button variant="outline">Edit Profile</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Edit profile</DialogTitle>
      <DialogDescription>
        Make changes to your profile here. Click save when you're done.
      </DialogDescription>
    </DialogHeader>
    <form className="grid gap-4 py-4">
      <div className="grid grid-cols-4 items-center gap-4">
        <label htmlFor="name" className="text-right">
          Name
        </label>
        <input
          id="name"
          defaultValue="Pedro Duarte"
          className="col-span-3 px-3 py-2 border rounded-md"
        />
      </div>
      <div className="grid grid-cols-4 items-center gap-4">
        <label htmlFor="username" className="text-right">
          Username
        </label>
        <input
          id="username"
          defaultValue="@peduarte"
          className="col-span-3 px-3 py-2 border rounded-md"
        />
      </div>
    </form>
    <DialogFooter>
      <Button type="submit">Save changes</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

### Controlled Dialog

```tsx
function ControlledDialog() {
  const [open, setOpen] = React.useState(false);

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button variant="outline">Open Controlled Dialog</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Controlled Dialog</DialogTitle>
          <DialogDescription>This is a controlled dialog.</DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button onClick={() => setOpen(false)}>Close</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Best Practices

- Use dialogs sparingly for important interactions that require immediate attention
- Keep dialog content concise and focused on a single task or piece of information
- Provide clear actions for the user to proceed or dismiss the dialog
- Consider the impact on mobile devices and ensure responsive design
- Use appropriate variants (e.g., alert dialogs) for different types of interactions

By following these guidelines and examples, you can effectively use and customize the Dialog component in your application to create accessible and user-friendly modal interactions.
