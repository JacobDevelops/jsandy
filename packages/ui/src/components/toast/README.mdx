# Toast Component System

## Overview

The Toast Component System is a comprehensive solution for displaying notifications in React applications. It provides a flexible and accessible way to show temporary messages to users, with features like customizable content, swipe-to-dismiss, and keyboard navigation.

## Features

- Customizable toast content (title, description, actions)
- Swipe-to-dismiss functionality
- Keyboard accessibility (Escape to close)
- Screen reader support
- Customizable duration and positioning
- Controlled and uncontrolled modes
- Styling with Tailwind CSS and class-variance-authority

## Installation

To use the Toast Component System, ensure you have the necessary dependencies installed in your project:

```bash
npm install @jsandy/ui
```

## Usage

Here's a basic example of how to use the Toast Component System:

```jsx
import {
  ToastProvider,
  ToastViewport,
  Toast,
  ToastTitle,
  ToastDescription,
  ToastAction,
  ToastClose,
} from '@jsandy/ui/toast';
import { useToast } from '@jsandy/ui/use-toast';

function App() {
  return (
    <ToastProvider>
      <YourApp />
      <ToastViewport />
    </ToastProvider>
  );
}

function YourApp() {
  const { toast } = useToast();

  const showToast = () => {
    toast({
      title: 'Scheduled: Catch up',
      description: 'Friday, February 10, 2023 at 5:57 PM',
      action: <ToastAction altText="Goto schedule to undo">Undo</ToastAction>,
    });
  };

  return <button onClick={showToast}>Show Toast</button>;
}
```

## API

### ToastProvider

The main wrapper component that provides context for the toast system.

#### Props

- `children`: React.ReactNode
- `duration`: number (default: 5000) - Default duration for toasts in milliseconds
- `swipeDirection`: 'up' | 'down' | 'left' | 'right' (default: 'right')
- `swipeThreshold`: number (default: 50) - Swipe distance threshold in pixels

### ToastViewport

Component that renders the container for toasts.

#### Props

- `className`: string - Additional CSS classes
- `hotkey`: string[] (default: ['F8']) - Keyboard shortcut to focus the viewport
- `label`: string (default: 'Notifications ({hotkey})') - Accessible label for the viewport

### Toast

Individual toast component.

#### Props

- `open`: boolean - Controlled open state
- `defaultOpen`: boolean - Default open state (uncontrolled)
- `onOpenChange`: (open: boolean) => void - Callback when open state changes
- `duration`: number - Custom duration for this toast
- `variant`: 'default' | 'destructive' - Visual variant of the toast

### ToastTitle

Component for rendering the toast title.

### ToastDescription

Component for rendering the toast description.

### ToastAction

Component for rendering an action button within a toast.

#### Props

- `altText`: string - Alternative text for accessibility

### ToastClose

Component for rendering a close button for a toast.

### useToast

Hook for programmatically creating and managing toasts.

#### Returns

- `toast`: (props: ToastProps) => void - Function to create a new toast
- `dismiss`: (toastId?: string) => void - Function to dismiss a toast

## Accessibility

The Toast Component System is designed with accessibility in mind:

- Proper ARIA attributes are used throughout
- Keyboard navigation is supported
- Screen reader announcements for new toasts
- Alternative text for actions

## Best Practices

1. Keep toast messages concise and informative
2. Use appropriate variants for different types of notifications
3. Provide clear actions when necessary
4. Don't overuse toasts - reserve them for important information
5. Consider the position of toasts to avoid covering important content
6. Test thoroughly with keyboard-only and screen reader users

## Customization

The Toast components can be customized using Tailwind CSS classes and the `className` prop. The `variant` prop on the Toast component allows for predefined style variations.
