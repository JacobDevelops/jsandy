# Table Component

## Overview

The Table component is a flexible and customizable way to display tabular data in React applications. It provides a structured layout for presenting information in rows and columns, with support for headers, footers, and various styling options.

## Features

- Responsive design with horizontal scrolling for small screens
- Customizable appearance through className props
- Support for complex table structures with header, body, and footer
- Hover and selected states for rows
- Accessible, using semantic HTML table elements
- Subcomponents for different parts of the table structure

## Installation

To use the Table component, ensure you have the necessary dependencies installed in your project:

```bash
npm install @jsandy/ui
```

## Usage

Here's a basic example of how to use the Table component:

```jsx
import {
  Table,
  TableHeader,
  TableBody,
  TableFooter,
  TableHead,
  TableRow,
  TableCell,
  TableCaption,
} from '@jsandy/ui/table';

function BasicTable() {
  return (
    <Table>
      <TableCaption>A list of your recent invoices</TableCaption>
      <TableHeader>
        <TableRow>
          <TableHead>Invoice</TableHead>
          <TableHead>Status</TableHead>
          <TableHead>Method</TableHead>
          <TableHead className="text-right">Amount</TableHead>
        </TableRow>
      </TableHeader>
      <TableBody>
        <TableRow>
          <TableCell className="font-medium">INV001</TableCell>
          <TableCell>Paid</TableCell>
          <TableCell>Credit Card</TableCell>
          <TableCell className="text-right">$250.00</TableCell>
        </TableRow>
        {/* Add more rows as needed */}
      </TableBody>
      <TableFooter>
        <TableRow>
          <TableCell colSpan={3}>Total</TableCell>
          <TableCell className="text-right">$250.00</TableCell>
        </TableRow>
      </TableFooter>
    </Table>
  );
}
```

## API

### Table

The main wrapper component for the entire table structure.

#### Props

Accepts all standard HTML table props.

### TableHeader

Component for grouping header content of a table.

#### Props

Accepts all standard HTML thead props.

### TableBody

Component for containing the main data rows of the table.

#### Props

Accepts all standard HTML tbody props.

### TableFooter

Component for the footer section of a table.

#### Props

Accepts all standard HTML tfoot props.

### TableRow

Component for a single row in the table.

#### Props

Accepts all standard HTML tr props.

### TableHead

Component for a header cell in a table.

#### Props

Accepts all standard HTML th props.

### TableCell

Component for a data cell in a table.

#### Props

Accepts all standard HTML td props.

### TableCaption

Component for providing a caption or summary for the table.

#### Props

Accepts all standard HTML caption props.

## Customization

All components accept a `className` prop for custom styling. They use Tailwind CSS classes by default, which can be overridden or extended.

## Accessibility

The Table component and its subcomponents use semantic HTML table elements, ensuring good accessibility by default. However, consider the following best practices:

1. Use TableCaption to provide a summary of the table contents.
2. Use TableHead components to clearly label columns.
3. For complex tables, consider using scope attributes on TableHead components.
4. Ensure sufficient color contrast for all text in the table.

## Best Practices

1. Use tables for presenting structured data that benefits from a tabular format.
2. Consider the amount of data and screen sizes when deciding to use a table.
3. Use TableHeader, TableBody, and TableFooter for proper table structure.
4. Provide clear column headers using TableHead components.
5. Use TableCaption to give context or summary of the table contents.
6. For large datasets, consider implementing pagination or virtual scrolling.
7. Use responsive design techniques for better mobile experience.
