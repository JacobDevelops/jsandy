# Tooltip Component System

## Overview

The Tooltip Component System is a flexible and accessible solution for displaying additional information when users hover over or focus on an element. It provides a customizable way to enhance user experience by offering contextual information without cluttering the interface.

## Features

- Customizable positioning (side, alignment, offsets)
- Delayed appearance for better user experience
- Keyboard accessibility
- Screen reader support
- Hover intent detection for nested tooltips
- Customizable styling

## Installation

To use the Tooltip Component System, ensure you have the necessary dependencies installed in your project:

```bash
npm install @jsandy/ui
```

## Usage

Here's a basic example of how to use the Tooltip Component System:

```jsx
import {
  TooltipProvider,
  Tooltip,
  TooltipTrigger,
  TooltipContent,
} from '@jsandy/ui/tooltip';

function App() {
  return (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger>Hover me</TooltipTrigger>
        <TooltipContent>This is a tooltip</TooltipContent>
      </Tooltip>
    </TooltipProvider>
  );
}
```

## API

### TooltipProvider

The main wrapper component that provides context for the tooltip system.

#### Props

- `delayDuration`: number (default: 700) - Delay before showing the tooltip
- `skipDelayDuration`: number (default: 300) - Skip delay when moving between tooltips
- `disableHoverableContent`: boolean (default: false) - Disable hovering on tooltip content

### Tooltip

The main component for creating a tooltip.

#### Props

- `open`: boolean - Controlled open state
- `defaultOpen`: boolean - Default open state (uncontrolled)
- `onOpenChange`: (open: boolean) => void - Callback when open state changes
- `delayDuration`: number - Custom delay for this tooltip
- `disableHoverableContent`: boolean - Disable hovering for this tooltip

### TooltipTrigger

Component for wrapping the element that triggers the tooltip.

### TooltipContent

Component for rendering the tooltip content.

#### Props

- `side`: 'top' | 'right' | 'bottom' | 'left' - Preferred side for the tooltip
- `sideOffset`: number - Offset from the trigger
- `align`: 'start' | 'center' | 'end' - Alignment along the side
- `alignOffset`: number - Offset for alignment
- `arrowPadding`: number - Padding between the arrow and the edges of the content
- `collisionBoundary`: Boundary - Element to use as boundary for collision detection
- `collisionPadding`: number | Padding - Space between tooltip and boundary
- `sticky`: 'partial' | 'always' - Sticky behavior on cross-axis
- `hideWhenDetached`: boolean - Hide tooltip when it's detached from the trigger

### TooltipArrow

Component for adding an arrow to the tooltip.

## Accessibility

The Tooltip Component System is designed with accessibility in mind:

- Proper ARIA attributes are used
- Keyboard navigation is supported
- Screen reader announcements for tooltip content

## Best Practices

1. Keep tooltip content concise and informative
2. Use tooltips for supplementary information, not critical content
3. Avoid nesting interactive elements inside tooltips
4. Consider touch devices when implementing tooltips
5. Use appropriate delay durations for your use case
6. Test tooltips with keyboard navigation and screen readers

## Customization

The Tooltip components can be customized using the `className` prop and Tailwind CSS classes. You can also use the `style` prop for inline styles.
