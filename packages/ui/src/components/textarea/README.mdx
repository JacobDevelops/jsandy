# Textarea Component

## Overview

The Textarea component is a customizable multi-line text input field for React applications. It provides a consistent and accessible way to collect longer text inputs from users while maintaining flexibility in styling and behavior.

## Features

- Customizable appearance through className prop
- Consistent styling with other form components
- Supports all standard textarea attributes
- Accessible focus states
- Disabled state styling
- Minimum height set for better usability

## Installation

To use the Textarea component, ensure you have the necessary dependencies installed in your project:

```bash
npm install @jsandy/ui
```

## Usage

Here's a basic example of how to use the Textarea component:

```jsx
import { Textarea } from '@jsandy/ui/textarea';

function Example() {
  return <Textarea placeholder="Enter your message here" rows={4} />;
}
```

## API

### Textarea

The main component for rendering a multi-line text input field.

#### Props

The Textarea component accepts all standard HTML textarea attributes. Some commonly used props include:

- `placeholder`: string
  - Specifies the placeholder text
- `rows`: number
  - Sets the visible number of lines for the textarea
- `disabled`: boolean
  - If true, disables the textarea
- `required`: boolean
  - If true, marks the textarea as required
- `className`: string
  - Additional CSS classes to apply to the textarea

## Customization

The Textarea component can be customized using the `className` prop. It uses Tailwind CSS classes by default, which can be overridden or extended.

## Accessibility

The Textarea component is designed with accessibility in mind:

- It uses semantic HTML (`<textarea>` element)
- It supports keyboard navigation
- It has visible focus states
- It can be easily associated with labels using the `id` attribute

## Best Practices

1. Always provide a label for the textarea, either using the `<label>` element or `aria-label` attribute.
2. Use placeholder text to provide hints, not to replace labels.
3. Consider setting an appropriate initial height using the `rows` attribute.
4. Allow for resizing unless there's a specific reason to restrict it.
5. Implement form validation for required fields.
6. Ensure sufficient color contrast for the input text and placeholder.
7. Provide clear error messages for invalid inputs.
